NXTCam Block

This block interfaces with NXTCam from mindsensors.com
At setup time, NXTCam allows you to define object(s) in it's environment by color.
At runtime, it can track object(s), and provide you it's color and bounding box.

If you have not already, please install NXTCamView program developed by Paul Tingey from sourceforge.net.
Tips and Tricks
In your NXT-G program, to begin tracking operation, you should add a block with "Enable Tracking" operation. This block should then be followed by a block to 'Get First Object'. At this time, from this block you can read the number of objects being tracked.

Use the Port Selector on the top of the block to make sure that the sensor port number (1, 2, 3, or 4) matches the port on the EV3 Brick that the NXTCam Sensor is connected to.

Use the Mode Selector to select the mode for the block. The Inputs and Outputs available will change depending on the mode.
Modes
Get First Object


Retrieves the number of objects tracked, as well as the color and coordinates of the first object.
Get nth Object


Allows the user to select the number of a specific object. Then retrieves the number of objects tracked, as well as the color and coordinates of the selected object.
Enable Tracking


Start tracking your objects. This energizes the camera chip (consuming power) and begins processing the images.
Disable Tracking


Stops tracking to conserve the battery power.
Line Tracking Mode


Enables Line tracking mode. Screen will display a broken line corresponding to each chosen color.
Object Tracking Mode


Enables Object tracking mode. Screen will display a rectangular box corresponding to each chosen color. (Default tracking mode)

Tips and Tricks
NXTCam captures an image, analyzes it, and extracts 8 objects from that image which will match the colors specified in the colormap.
Before returning the object info to you (coordinates and color), this command specifies how to sort them.
They are sorted first to last.
Sort Objects by Size


The objects tracked could be of varying sizes. This option sorts them by the size while processing the image. The largest object will be first object.
Sort Objects by Colors


The objects tracked could be of varying colors. This option sorts them by color while processing the image. Color 1 (selected in nxtcamview) will first color.
Inputs and Outputs
The inputs available for the NXTCam sensor block will depend on the mode selected. You can enter the input values directly into the block. Alternatively, the input values can be supplied by Data Wires from the outputs of other Programming Blocks.
Input Type Allowed Values Notes
I2C Address Numeric Any number Establishes connection to EV3 brick
The I2C address of the sensor.  The factory default address is 2 (0x02). Choose appropriate value if you have specifically changed the address.
Object Number to Track Numeric 0 - 8 Allows the user to specify which number object to track.
The outputs available will depend on the mode selected. To use an output, use a Data Wire to connect it to another Programming Block.
Output Type Notes
Total Objects Tracked Numeric Total number of objects tracked (0-8).
Color Numeric Numeric value of color in colormap (0-8).
X Upper Left Numeric Upper left X-coordinate of object. X-coordinates range 0-144.
Y Upper Left Numeric Upper left Y-coordinate of object. Y-coordinates range 0-88
X Lower Right Numeric Lower right X-coordinate of object.
Y Lower Right Numeric Lower right of Y-coordinate of object.
Success? Logic Allows the user to test the output data for read errors.
